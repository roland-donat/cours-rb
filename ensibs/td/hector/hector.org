* Introduction
Hector, le robot jongleur, lâche souvent les balles avec lesquelles il jongle quand sa batterie est
faible. 

D'après les expériences précédentes, il a été déterminé que :
- la probabilité qu'il lâche une balle quand sa batterie est faible est de $0.9$ ;
- quand sa batterie n'est pas faible, la probabilité qu'il lâche une balle est seulement de $0.01$ ;
- quand la batterie a été rechargée il y a peu de temps, il y a seulement $5\%$ de chances que la
  batterie soit faible contre $50\%$ si la batterie n'a pas été rechargée récemment. 

Un premier système de vision $O_{1}$
(peu fiable) observe le robot et nous prévient lorsqu'il croit qu'Hector a lâché une balle. Un
autre système $O_{2}$ (indépendant du premier) agit de la même façon. 

Le but de cet exercice est de
modéliser les résultats des observateurs en fonction de ce que fait Hector et de son état.

* Structure du réseau bayésien

1. En supposant que toutes les variables soient booléennes, quelles variables allez-vous choisir pour
  modéliser ce problème ?
2. Proposer une structure de réseau bayésien correspondant au problème. Étiqueter les noeuds du réseau et indiquer

3. Donner l'expression de la loi de probabilité jointe associée au RB construit précédemment
4. D'après votre modélisation, les résultats des deux observateurs sont-ils réellement indépendants ?
   À quelle(s) condition(s) y a-t-il indépendance ? 

* Lois de probabilité conditionnelles

1. À partir des informations disponibles, représenter les LPC correspondant à votre structure. Il
   vous manquera des données pour finir le travail, lesquelles ? 
2. Calculer la complexité du RB et comparer la avec la complexité de la loi jointe du modèle
   représentée naïvement (i.e. sans RB).

* Implémentation et exploitation du modèle

On suppose que la fiabilité des observateurs $O_{1}$ et $O_{2}$ est de respectivement
$70\%$ et $90\%$.

L'objectif est de répondre aux trois questions suivantes en utilisant la librairie =PyAgrum=, puis
le logiciel =GeNIe=.

1. Calculer la probabilité a priori de chacun des noeuds du réseau.
2. $O_{1}$ observe que Hector a lâché une balle. Quelle est la probabilité que la batterie soit
   faible sachant cela? Comparer cette valeur à la probabilité /a priori/ que la batterie soit faible.
3. On ajoute alors une information supplémentaire : $O_{2}$ n'a rien vu (à la différence de
   $O_{1}$). Quelle est la probabilité que la batterie soit faible sachant ces deux informations ?
   Comparer aux résultats précédents.

** Utilisation de la librairie =PyAgrum=

Pour utiliser la librairie =Python= =PyAgrum=, il faut dans un premier temps l'installer dans votre
environnement. Notons que =PyAgrum= fonctionne dans n'importe quel environnement
=Python=. Toutefois, certaines fonctions de visualisation ne sont disponibles que dans les
environnements de type /notebook/ (e.g. Jupyter). De plus, ces fonctionnalités de visualisation
nécessite l'installation de l'outil =graphviz= (sous Linux/Ubuntu, =sudo apt install graphviz=). 

*** Création de la structure du modèle

1. L'installation de la librairie se fait classiquement /via/ la commande =!pip install <nom_module>= : 
   #+BEGIN_SRC python :eval no :exports code 
!pip install pyagrum
   #+END_SRC
   Puis importer le module :
   #+BEGIN_SRC python :session :results table :tangle hector.py :exports both
import pyAgrum as gum # La librairie pyAgrum

# Vérification des versions
{
    "pyagrum": gum.__version__, 
}
   #+END_SRC

   #+RESULTS:
   | pyagrum | : | 1.7.1 |

1. Nous allons à présent initialiser notre réseau bayésien grâce à la fonction =gum.BayesNet= :
   #+BEGIN_SRC python :session :results value :tangle hector.py :exports both
bn = gum.BayesNet("Hector")
bn
   #+END_SRC

   #+RESULTS:
   : BN{nodes: 0, arcs: 0, domainSize: 1, dim: 0, mem: 0o}

   En analysant l'objet =bn=, vous constatez que votre RB ne contient ni variable (/nodes/), ni
   relation de dépendance (/arcs/).
   
1. L'étape suivante consiste à créer les cinq variables aléatoire (v.a.) discrètes et finies du modèle avec la fonction
   =gum.LabelizedVariable=. Commençons avec la variable $C$ indiquant si Hector a été rechargé
   récemment ou non. La fonction =LabelizedVariable= prend en argument un identifiant unique pour la
     variable, une description optionnelle et un nombre de modalités (par défaut 2) :
   #+BEGIN_SRC python :session :results value :tangle hector.py :exports both
va_C = gum.LabelizedVariable("C", "Charge récente", 2)
va_C
   #+END_SRC

   #+RESULTS:
   : C:Labelized({0|1})

   On remarque que la variable créée est bien binaire avec pour modalités 0 et 1.

   Poursuivons avec la définition des autres variables :
   #+BEGIN_SRC python :session :results value :tangle hector.py :exports both
va_B = gum.LabelizedVariable("B", "Batterie ok ?", 2)
va_L = gum.LabelizedVariable("L", "Hector lâche une balle ?", 2)
va_O1 = gum.LabelizedVariable("O1", "O1 : chute observée ?", 2)
va_O2 = gum.LabelizedVariable("O2", "O2 : chute observée ?", 2)
   #+END_SRC

   #+RESULTS:

   Il est également possible de changer le nom des modalités de variable avec la méthode
   =.changeLabel=. Par exemple, nous pouvons changer le nom de la première modalité par =non= et le nom de la seconde
   modalité par =oui= pour toutes les variables :
   #+BEGIN_SRC python :session :results value :tangle hector.py :exports both
for va in [va_C, va_B, va_L, va_O1, va_O2]:
    va.changeLabel(0, "non")
    va.changeLabel(1, "oui")
va_C
   #+END_SRC
   
   #+RESULTS:
   : C:Labelized({non|oui})

1. Il s'agit à présent d'ajouter les variables précédemment créées dans le RB grâce à la méthode

for va in [va_C, va_B, va_L, va_O1, va_O2]:
    bn.add(va)
bn
   #+END_SRC

*** Définition des LPC
    
1. La structure du RB est prête, il faut maintenant renseigner les lois de probabilité
   conditionnelles (LPC) de chaque variable. Pour une variable donnée, on renseigne la loi de
   probabilité associée à chaque configuration de modalités de ses variables parentes.

   Pour les variables $C$ et $B$, cela donne :
   #+BEGIN_SRC python :session :results html :tangle hector.py :exports both
# Pas de parent, on renseigne directement la loi (a priori) de la variable
bn.cpt("C")[:] = [0.5, 0.5]

bn.cpt("B")[{"C": "non"}] = [0.5, 0.5]

*** Inférence

1. Le RB est entièrement défini. Nous pouvons donc exploiter notre modèle afin de répondre aux
   questions de l'exercice. Calculer les probabilités /a priori/ de chaque variable consiste à
   calculer les lois $P(C)$, $P(B)$, $P(L)$, $P(O_{1})$ et $P(O_{2})$. Pour réaliser ces calculs, il
   faut définir un moteur d'inférence (i.e. une machine à calculer les probabilités pour notre
   RB). Dans cet exemple, nous utilisons la méthode appelée /lazy propagation/ qui est une technique
   de calculs probabilistes exacts relativement sophistiquée utilisant les principes de la méthode
   d'élimination que nous étudierons dans ce module.

   Commençons par initialiser le moteur d'inférence avec la fonction =gum.LazyPropagation= :
   #+BEGIN_SRC python :session :results value :tangle hector.py :exports both
bn_ie = gum.LazyPropagation(bn) # ie pour inference engine
   #+END_SRC

   #+RESULTS:

1. Il faut ensuite lancer les calculs de probabilité dans le RB avec la méthode =.makeInference= :
   #+BEGIN_SRC python :session :results silent :tangle hector.py :exports both

   #+END_SRC

   - La fonction =showInference= gère l'initialisation du moteur d'inférence. Il n'est donc pas
     nécessaire de créer un moteur d'inférence si vous souhaitez uniquement utiliser
     =showInference=.
   - N'hésitez pas à consulter l'[[https://pyagrum.readthedocs.io/en/1.7.1/lib.notebook.html#pyAgrum.lib.notebook.showInference][aide de la fonction =showInference=]] afin d'appréhender ces
     différentes fonctionnalités.

1. La seconde question consiste à calculer la probabilité de l'état de la batterie d'Hector sachant que l'observateur $O_{1}$ a observé un lâché de balle. Il s'agit autrement dit de
   calculer la loi conditionnelle $P(B|O_{1}=\text{oui})$. Il faut donc conditionner le moteur
   d'inférence avec l'information
   $O_{1}=\text{oui}$ grâce à la méthode =.setEvidence=, puis refaire les calculs :
   #+BEGIN_SRC python :session :results silent :tangle hector.py :exports code
bn_ie.setEvidence({"O1":"oui"})
bn_ie.makeInference()
P_B_sachant_O1_oui = bn_ie.posterior("B")
P_B_sachant_O1_oui
   #+END_SRC
   #+BEGIN_SRC python :session :results html :tangle hector.py :exports results
gnb.getPotential(P_B_sachant_O1_oui)
   #+END_SRC
   *Attention :* : La loi précédente correspond bien à $P(B|O_{1}=\text{oui})$ et non à $P(B|O_{1})$.
   

   Alternativement, on peut directement utiliser la méthode =showInference= en précisant le
   conditionnement à appliquer /via/ l'argument =evs=. 
   #+BEGIN_SRC python :session :results silent :tangle hector.py :exports code
gnb.showInference(bn, evs={"O1":"oui"})
   #+END_SRC
   #+BEGIN_SRC python :session :results html :tangle hector.py :exports results
gnb.getInference(bn, evs={"O1":"oui"})
   #+END_SRC

2. La dernière question consiste à calculer la probabilité de l'état de la batterie d'Hector sachant
   que l'observateur $O_{1}$ a observé un lâché de balle mais que l'observateur $O_{2}$ n'a rien vu
   d'anormal. Il s'agit donc de
   calculer la loi conditionnelle $P(B|O_{1}=\text{oui}, O_{2}=\text{non})$. Calculez cette loi avec
   les deux méthodes précédentes. Pour le calcul avec =makeInference=, supprimer auparavant les
   éventuels conditionnements précédents avec la méthode =.eraseAllEvidence= :
   #+BEGIN_SRC python :session :results value :tangle hector.py :exports code
bn_ie.eraseAllEvidence()
   #+END_SRC

   #+BEGIN_SRC python :session :results silent :tangle hector.py :exports (if (eq td_corrige t) "code" "none")
# SOLUTION
# --------
bn_ie.eraseAllEvidence()
bn_ie.setEvidence({"O1":"oui", "O2":"non"})
bn_ie.makeInference()
P_B_sachant_O1_oui_O2_non = bn_ie.posterior("B")
P_B_sachant_O1_oui_O2_non
   #+END_SRC
   #+BEGIN_SRC python :session :results html :tangle hector.py :exports (if (eq td_corrige t) "results" "none")
gnb.getPotential(P_B_sachant_O1_oui_O2_non)
   #+END_SRC

   De la même façon, en utilisant =gnb.showInference= :
   #+BEGIN_SRC python :session :results silent :tangle hector.py :exports (if (eq td_corrige t) "code" "none")
# SOLUTION
# --------
gnb.showInference(bn, evs={"O1":"oui", "O2":"non"})
    #+END_SRC
   #+BEGIN_SRC python :session :results html :tangle hector.py :exports (if (eq td_corrige t) "results" "none")
gnb.getInference(bn, evs={"O1":"oui", "O2":"non"})

** Utilisation du logiciel =GeNIe=

L'objectif est à présent de refaire le travail précédent mais cette fois-ci à partir du logiciel =GeNIe=.

Rappelons que =GeNIe= est une interface graphique permettant de construire des RB. Pour télécharger
cet outil, il faut se rendre sur le site de [[http://www.bayesfusion.com/#!downloads/rxu9q][BayesFusion, LLC]] et cliquer sur le 
lien associé à la version académique.

Pour les utilisateurs de =Linux=, =GeNIe= fonctionne bien sous =Wine=.

** Interropérabilité entre =pyAgrum= et =GeNIe=                    :noexport:

NE MARCHE PAS POUR L'INSTANT : DATE : 05/2023

Le logiciel =GeNIe= sauvegarde les RB sous le format =xdsl= (qui est en fait du =XML=). Or,

et à la méthode =.saveXDSL=. 

#+BEGIN_SRC python :session :results silent :tangle hector.py :exports code :eval no
bn.saveXDSL("hector_bis.xdsl")
#+END_SRC

bn_bis = bn.loadXDSL("ex_hector.xdsl")
#+END_SRC
